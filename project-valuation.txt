FACEBOOK MARKET TOOL - PROJECT VALUATION
Date: 2026-05-12

Purpose
This document estimates how much the current project is worth as a software asset.
It is based on the repository's visible scope, code volume, integrations, testing,
deployment evidence, and technical risk profile.

This is not a revenue multiple or investor valuation.
It is an engineering and asset valuation based on replacement cost and current
fair market value for a custom internal business system.


1. EXECUTIVE VALUATION SUMMARY

Recommended current project value: $75,000

Reasonable fair-market range: $60,000 to $95,000

Estimated rebuild / replacement cost: $110,000 to $140,000

Quick-sale / asset-only floor: $25,000 to $40,000

Best interpretation:
- If someone needed this exact system rebuilt for internal use, they would likely
  spend around $120,000.
- Because the current codebase still carries platform and hardening risk, its
  present fair value as an asset is lower than rebuild cost.
- A practical headline number for negotiations is $75,000, with supportable room
  to argue into the $85,000 to $95,000 range if the buyer strongly values the
  Salesforce + Apps Script + Google Sheets workflow that already exists.

3. VALUATION METHOD USED

I used three valuation lenses:

Method A - Replacement cost
What it would likely cost to rebuild this to comparable functionality using a
competent contractor or small product team.

Method B - Current fair market value
What the existing codebase is reasonably worth today, after discounting for
technical debt, platform concentration, and incomplete roadmap items.

Method C - Quick-sale floor
What the code and documentation may fetch if sold fast as a useful internal tool
template without deep transition support.

Because there is no revenue, profit, customer contract list, or active user-count
history in the repo, I am not applying SaaS multiples, DCF, or ARR-based methods.

5. WHY THE CURRENT FAIR VALUE IS LOWER THAN REBUILD COST

Software assets usually sell below full rebuild cost unless they have strong
recurring revenue, a large user base, or unusually portable IP.

Current value discounts visible from the repo:

1) Platform concentration risk
- Core operations still depend heavily on Apps Script, Sheets, and one execution
  identity/quota pool.
- This lowers portability and caps scalability.

2) Security and hardening gaps still documented by the repo itself
- Review documents call out open deployment exposure, signature hardening needs,
  and operational risk around public endpoints and replay protection.

3) Maintainability debt
- Large single-file surfaces still exist in the Apps Script code and HTML pages.
- That raises future maintenance cost for any buyer.

4) Some roadmap value is still unrealized
- The repo contains blueprint material for a more advanced Facebook push flow,
  but that appears planned rather than fully shipped end-to-end.

5) Niche buyer pool
- This is a business-specific internal workflow tool, not a broad consumer app.
- That reduces how many buyers can use it without adaptation.


6. WHAT SUPPORTS A STRONGER VALUE

This project still deserves a meaningful valuation because it already solves a
real business workflow and contains non-trivial integration work.

Value-supporting strengths:

1) Production orientation is real
- The repo includes live Apps Script deployment workflow, Salesforce deployment
  workflow, sandbox refresh tooling, and versioned historical references.

2) Dual-platform integration is expensive to reproduce
- Salesforce + Apps Script + Google Sheets + OpenAI is a specialized stack.
- Signed embedding and secure server-to-server call patterns add real value.

3) It is more than a prototype
- There are tests, docs, deployment scripts, architecture reviews, and current
  operational guidance in the repo.

4) The business workflow is specific and already modeled
- Salesperson flows, listings management, analytics, manager view behavior, and
  inventory-related operations are already encoded.

5) Migration leverage exists
- Even if a future team moves off Apps Script, this repo still shortens the path
  because business rules, UI flow, sheet schema assumptions, and Salesforce data
  exchange patterns are already documented and implemented.


7. CURRENT FAIR-MARKET VALUE ESTIMATE

My recommended present-value view is:

- Conservative fair value: $60,000
- Most likely fair value: $75,000
- Strong strategic-buyer case: $95,000

How that number was formed:
- Start from a replacement value around $120,000
- Discount 20% to 35% for platform risk, technical debt, and limited portability
- Retain value because the system is already operationally useful and materially
  reduces the time to deploy a working internal listing workflow

That produces a realistic current value band of roughly $60,000 to $95,000.

10. WHAT WOULD INCREASE THE VALUATION FASTEST

The following improvements would make the project easier to defend at a higher
value:

1) Prove current business usage
- Active users, listings generated per month, time saved, or sales lift

2) Close the highest-risk hardening gaps
- Restrict anonymous exposure, tighten signature model, reduce replay risk,
  reduce hidden iframe fan-out load

3) Ship the highest-value planned features fully
- If Facebook push, queue controls, and stronger observability become fully live,
  strategic value increases

4) Reduce platform concentration risk
- Separate critical backend functions from the most fragile Apps Script limits

5) Add stronger test and operational evidence
- Coverage growth, runbooks, monitoring, alerting, and deployment history all
  make the asset easier for another team to trust


11. BOTTOM LINE

This project is not just a script bundle. It is a functioning internal software
asset with:
- a custom Google Apps Script application
- a Salesforce integration package
- AI-assisted listing generation
- spreadsheet-backed operations
- deployment workflow
- tests and architecture documentation

On a pure engineering replacement basis, it looks like a low-six-figure build.
On a current asset basis, a realistic present value is about $75,000, with a
reasonable range of $60,000 to $95,000 depending on buyer fit and whether the
valuation is framed as a rebuild alternative or as a fast-transfer code asset.
